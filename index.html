The recommended setup is based on a conda environment. The Makefiles then resolve most third-party libraries through CONDA_PREFIX, while Trilinos/Sacado is expected under $CONDA_PREFIX/trilinos-serial unless another path is provided.

Keep installation files in GitHub, and use this page as the guide.

Theory

From Neo-Hookean energy to AD residual and tangent.

The hyperelastic model is state-based: once the displacement field defines the deformation gradient, the stress follows from the strain-energy density. The element weak form is then written once as a scalar statement and differentiated by AD to obtain residual and tangent.

1 · Kinematics

Deformation gradient

\[ \mathbf{F} = \mathbf{I} + \nabla_0 \mathbf{u} \]

The displacement field is interpolated from the element DOFs. At every Gauss point, the reference gradient of the displacement defines the finite-strain deformation gradient.

2 · Material law

Compressible Neo-Hookean model

\[ J = \det \mathbf{F}, \qquad \mathbf{C} = \mathbf{F}^{T}\mathbf{F} \]
\[ W(\mathbf{F}) = \frac{\mu}{2}\left(\operatorname{tr}(\mathbf{C}) - d\right) - \mu \ln J + \frac{\lambda}{2}(\ln J)^2 \]

Young’s modulus and Poisson’s ratio are read from Material.txt and converted to Lamé parameters before evaluating the stress response.

3 · Parameters

Input constants to Lamé constants

\[ \mu = \frac{E}{2(1+\nu)}, \qquad \lambda = \frac{E\nu}{(1+\nu)(1-2\nu)} \]

For this example, the material file uses youngs_modulus 200 and poisson_ratio 0.3.

4 · Stress

First Piola stress

\[ \mathbf{P} = \frac{\partial W}{\partial \mathbf{F}} \]

Because the weak form is integrated over the reference domain, the implementation uses the first Piola-Kirchhoff stress.

5 · Weak form

Element scalar declaration

\[ G_e(\mathbf{u}, \delta\mathbf{u}) = \int_{\Omega_0^e} \mathbf{P}(\mathbf{F}(\mathbf{u})) : \nabla_0 \delta\mathbf{u}\, dV - \int_{\Gamma_{t,e}} \bar{\mathbf{t}} \cdot \delta\mathbf{u}\, d\Gamma \]

This is the central programming idea: the model declares the local weak-form contribution, while the global assembly and solver remain independent of the material model.

6 · AD residual

Differentiate with respect to virtual displacement

\[ \mathbf{R}_{e,i} = \frac{\partial G_e}{\partial (\delta u_i)} \]

The virtual displacement variables are AD variables, so each element residual entry is read from the derivative of the scalar weak-form expression.

7 · AD tangent

Differentiate residual with respect to current DOFs

\[ \mathbf{K}_{e,ij} = \frac{\partial \mathbf{R}_{e,i}}{\partial u_j} \]

The current displacement DOFs are wrapped by an outer AD type, which gives the Newton tangent from the same implementation used to compute the residual.

template <class Scalar>
inline std::vector<std::vector<Scalar>> kinematic_embed_3d(
    const std::vector<std::vector<Scalar>>& A,
    int dim
) {
    if (dim == 3) return A;

    auto A3 = kinematic_identity<Scalar>(3);
    for (int i = 0; i < 2; ++i) {
        for (int j = 0; j < 2; ++j) {
            A3[i][j] = A[i][j];
        }
    }
    return A3;
}
4. Neo-Hookean first Piola helper 
template <class Scalar>
inline std::vector<std::vector<Scalar>> kinematic_neo_hookean_first_piola_lame(
    const std::vector<std::vector<Scalar>>& F3,
    double lambda,
    double mu
) {
    const Scalar J = kinematic_det(F3);
    const auto FinvT = kinematic_inv_transpose(F3);
    const Scalar coeff = Scalar(0.5 * lambda) * (J * J - Scalar(1.0));

    std::vector<std::vector<Scalar>> P(3, std::vector<Scalar>(3, Scalar(0.0)));
    for (int i = 0; i < 3; ++i) {
        for (int A = 0; A < 3; ++A) {
            P[i][A] = Scalar(mu) * (F3[i][A] - FinvT[i][A])
                    + coeff * FinvT[i][A];
        }
    }
    return P;
}

Solid Mechanics / Inelastic / Elastoplasticity

Elastoplasticity with local history variables

This page explains the finite-strain elastoplastic update used by the solid weak form. The global problem remains displacement-based, while plastic variables are solved locally at each Gauss point.

Core idea

Global displacement solve, local plastic return mapping.

Hyperelasticity is state-based: once the deformation gradient \(\mathbf{F}\) is known, the stress follows directly. Elastoplasticity is path-dependent. The current stress depends on the accepted Gauss-point history from the previous load step and on a local constitutive update performed during the weak-form evaluation.

Flow

displacement DOFs → \(\mathbf{F}\) → local solve \(\mathbf{Q}(\mathbf{F},\mathbf{h})=0\) → stress \(\mathbf{P}\) → consistent tangent → global Newton assembly

Input declaration

The history declaration defines the local state used by the theory.

The global finite element problem contains only displacement DOFs. Plasticity enters through Gauss-point history variables, which are updated locally and committed only after a converged step.

History vector

The local history vector is written as

\[ \mathbf{h} = \begin{bmatrix} h_{11} & h_{22} & h_{33} & h_{12} & \gamma \end{bmatrix}^{T}. \]
  • h11, h22, h33, h12 define the plastic metric-like internal tensor.
  • gamma is the accumulated plastic variable used by the hardening law.
  • h_n is the last accepted state; h_trial is the current Newton trial state.

Plastic metric from history

\[ \mathbf{C}_{gp}^{-1}(\mathbf{h}) = \begin{bmatrix} 1+h_{11} & h_{12} & 0 \\ h_{12} & 1+h_{22} & 0 \\ 0 & 0 & 1+h_{33} \end{bmatrix}. \]

For a two-dimensional plane-strain formulation, the out-of-plane component is still retained so that the constitutive update is performed in a three-dimensional tensor setting.

Kinematics

Elastic tensor

The global displacement field gives the deformation gradient

\[ \mathbf{F}=\mathbf{I}+\nabla_{0}\mathbf{u}. \]

The elastic tensor used by the stress model is

\[ \mathbf{b}_{e} = \mathbf{F}\mathbf{C}_{gp}^{-1}\mathbf{F}^{T}. \]

This is the first point where the local history affects the current stress. If \(\mathbf{h}=\mathbf{0}\), then \(\mathbf{C}_{gp}^{-1}=\mathbf{I}\) and the update reduces to the elastic trial state.

Stress check

Kirchhoff stress and yield function

\[ \boldsymbol{\tau}=\boldsymbol{\tau}(\mathbf{b}_{e}), \qquad \Phi(\boldsymbol{\tau},\gamma) = q(\boldsymbol{\tau})-\sigma_{y}(\gamma). \]

If \(\Phi_{trial}\leq 0\), the step is elastic and the accepted history is copied to the trial state. If \(\Phi_{trial}>0\), the local plastic residual must be solved.

\[ \sigma_{y}(\gamma) = \sigma_{y0} +K_f\left(1-\exp(-\delta\gamma)\right) +Y_{\infty}\gamma. \]
Exponential mapping

The plastic update is written as a local tensor equation.

Let \(\mathbf{h}_{n}\) be the accepted history from the previous converged step and \(\mathbf{h}\) be the unknown trial history at the current Gauss point. The plastic increment is \(\Delta\gamma=\gamma-\gamma_{n}\), and the flow direction is evaluated from the current stress as \(\mathbf{A}=\mathbf{A}(\boldsymbol{\tau})\).

The exponential map evolves the plastic metric through

\[ \exp(-2\Delta\gamma\mathbf{A}). \]

The tensor residual is defined as

\[ \mathbf{Z}(\mathbf{F},\mathbf{h}) = \mathbf{F}\mathbf{C}_{gp}^{-1}(\mathbf{h}) - \exp(-2\Delta\gamma\mathbf{A}) \mathbf{F}\mathbf{C}_{gp,n}^{-1}. \]

The five-component local residual is

\[ \mathbf{Q}(\mathbf{F},\mathbf{h}) = \begin{bmatrix} Z_{11} \\ Z_{22} \\ Z_{33} \\ Z_{12} \\ \Phi \end{bmatrix} = \mathbf{0}. \]
Local Newton

The material point is solved before element assembly.

\[ \mathbf{J}_{h} = \frac{\partial\mathbf{Q}}{\partial\mathbf{h}}, \qquad \mathbf{J}_{h}\Delta\mathbf{h} = -\mathbf{Q}. \] \[ \mathbf{h}^{k+1} = \mathbf{h}^{k}+\alpha\Delta\mathbf{h}. \]

A line-search factor \(\alpha\) can be used to keep the local update stable, for example by avoiding nonphysical plastic metric states.

Consistent tangent

The global tangent includes the sensitivity of the local solve.

\[ \frac{d\mathbf{h}}{d\mathbf{F}} = - \mathbf{Q}_{,\mathbf{h}}^{-1} \mathbf{Q}_{,\mathbf{F}}. \] \[ \mathbb{A}_{ep} = \frac{d\mathbf{P}}{d\mathbf{F}} = \mathbf{P}_{,\mathbf{F}} - \mathbf{P}_{,\mathbf{h}} \mathbf{Q}_{,\mathbf{h}}^{-1} \mathbf{Q}_{,\mathbf{F}}. \]

This is the key difference between a secant-like plastic update and a consistent Newton update.

Weak form

The global solver still sees only residual and tangent.

\[ G(\mathbf{u},\delta\mathbf{u}) = \int_{\Omega_0} \mathbf{P}(\mathbf{F},\mathbf{h}(\mathbf{F})) :\nabla_{0}\delta\mathbf{u}\,dV - \int_{\Gamma_N} \bar{\mathbf{t}}\cdot\delta\mathbf{u}\,dA. \] \[ \mathbf{R}_{e} = \frac{\partial G_e}{\partial\delta\mathbf{u}}, \qquad \mathbf{K}_{e} = \frac{\partial\mathbf{R}_{e}}{\partial\mathbf{u}}. \]

static Vector local_Qg_residual(const Mat3& F,const Vector& h_n,
                                const Vector& h,const EPMaterial& mat){

    const Mat3 Cgpin=Cgpi_from_h(h_n);

        kinematic_mul3<double>(F,Cgpi), kinematic_transpose3<double>(F));
    const Mat3 tau=tau_from_be(be,mat);
    const double Phi=yield_q(tau)-sigma_y(gamma_from_h(h),mat);
    const Mat3 A=yield_A(tau);
    const double dgam=gamma_from_h(h)-gamma_from_h(h_n);
    const Mat3 M=kinematic_scale3<double>(A,-2.0*dgam);

        kinematic_mul3<double>(F,Cgpi),
        kinematic_mul3<double>(kinematic_mul3<double>(expm3(M),F),Cgpin));
    return {Z[0][0],Z[1][1],Z[2][2],Z[0][1],Phi};
}
4. Local return mapping: double-valued Newton solve at one Gauss point. 
static PlasticUpdateResult update_plastic_gp(const Mat3& F,const Vector& h_n,
                                             double state_n,const EPMaterial& mat){
    (void)state_n;
    PlasticUpdateResult out; out.h=h_n;
    const double Ftrial=yield_function(F,h_n,mat);
    const double tol=std::max(mat.local_tol,1e-12);
    if(Ftrial<=tol){
        out.plastic=false; out.final_local_residual=0.0;
        out.local_newton_iterations=0; out.state_trial=0.0;
        return out;
    }
    out.plastic=true;
    Vector h=h_n;
    for(int it=0; it<std::max(1,mat.local_max_iter); ++it){
        Vector Q=local_Qg_residual(F,h_n,h,mat);
        Matrix J=local_Jh_sacado(F,h_n,h,mat);
        Vector rhs(Q.size());
        for(int i=0;i<static_cast<int>(Q.size());++i) rhs[i]=-Q[i];
        Vector dh=solve_dense(J,rhs);

        double alpha=1.0;
        const double q0=norm_vec(Q);
        for(int ls=0; ls<8; ++ls){
            Vector ht=h;
            for(int i=0;i<5;++i) ht[i]+=alpha*dh[i];
            if(kinematic_det3<double>(Cgpi_from_h(ht))>0.0){
                Vector Qt=local_Qg_residual(F,h_n,ht,mat);
                if(norm_vec(Qt)<=q0 || alpha<1e-3){ h=ht; break; }
            }
            alpha*=0.5;
        }
        Vector Qnew=local_Qg_residual(F,h_n,h,mat);
        const double qn=norm_vec(Qnew);
        if(qn<tol || norm_vec(dh)<tol){
            out.h=h; out.final_local_residual=qn;
            out.local_newton_iterations=it+1;
            out.state_trial=1000.0+qn;
            return out;
        }
    }
    throw std::runtime_error("Local elasto-plastic Newton did not converge.");
}

    Matrix Q_F(nh, Vector(nf, 0.0));
    Matrix Q_h(nh, Vector(nh, 0.0));
    for (int i = 0; i < nh; ++i) {
        for (int j = 0; j < nf; ++j) Q_F[i][j] = Q[i].dx(j);
        for (int j = 0; j < nh; ++j) Q_h[i][j] = Q[i].dx(nf + j);

Theory

Finite-strain viscoelastic update from a local metric equation.

The unknown solved by the global Newton method is the displacement field. The viscous internal variable is stored locally as a symmetric tensor \(\mathbf{C}_v\). At each Gauss point, the current deformation gradient \(\mathbf{F}\) and the previous accepted history \(\mathbf{C}_{v,n}\) define a nonlinear local residual \(\mathbf{Q}=\mathbf{0}\). Solving this local problem gives the updated history and the algorithmic stress used in the global weak form.

1 · Kinematics

Current deformation and elastic left metric

\[ \mathbf{F}=\mathbf{I}+\nabla_0\mathbf{u}, \qquad \mathbf{C}=\mathbf{F}^{T}\mathbf{F} \]
\[ \mathbf{b}_e=\mathbf{F}\mathbf{C}_v^{-1}\mathbf{F}^{T} \]

The total deformation is measured by \(\mathbf{F}\), while the viscous metric \(\mathbf{C}_v\) separates the elastic part through \(\mathbf{b}_e\). Initially, the history is zero, meaning \(\mathbf{C}_v=\mathbf{I}\).

2 · History variables

Stored Gauss-point metric

\[ \mathbf{C}_v(\mathbf{h})= \begin{bmatrix} 1+h_{11} & h_{12} & 0 \\ h_{12} & 1+h_{22} & 0 \\ 0 & 0 & 1+h_{33} \end{bmatrix} \qquad \text{plane strain} \]
\[ \mathbf{h}_{2D}=\begin{bmatrix} h_{11} \\ h_{22} \\ h_{33} \\ h_{12} \end{bmatrix} = \begin{bmatrix} C_{v,11}-1 \\ C_{v,22}-1 \\ C_{v,33}-1 \\ C_{v,12} \end{bmatrix} \]

The website shows the history in tensor form first, matching the plasticity page style. The code stores the independent entries as a compact Gauss-point vector only for memory layout. At initialization \(\mathbf{h}=\mathbf{0}\), so \(\mathbf{C}_v=\mathbf{I}\).

3 · Stress

Kirchhoff stress from the elastic metric

\[ \boldsymbol{\tau}=2\mathbf{b}_e\frac{\partial \psi(\mathbf{b}_e)}{\partial \mathbf{b}_e} \]
\[ \boldsymbol{\tau}=p\mathbf{I}+\mu J_b^{-1/3} \left(\mathbf{b}_e-\frac{\operatorname{tr}(\mathbf{b}_e)}{3}\mathbf{I}\right), \qquad p=\frac{\kappa}{2}(J_b-1) \]

The model uses a compressible Neo-Hookean-type Kirchhoff stress, but it is evaluated on the elastic metric \(\mathbf{b}_e\), not directly on the total \(\mathbf{F}\mathbf{F}^T\).

4 · Local residual

Viscous flow equation

\[ \dot{\mathbf{C}}_v\approx\frac{\mathbf{C}_v-\mathbf{C}_{v,n}}{\Delta t} \]
\[ \mathbf{d}_v=\operatorname{sym}\left( \frac{1}{2}\mathbf{F}^{-T}\dot{\mathbf{C}}_v\mathbf{C}^{-1}\mathbf{F}^{T} \right) \]
\[ \mathbf{Q}=-\operatorname{dev}\left(\boldsymbol{\tau}-\eta\mathbf{d}_v\right), \qquad \det(\mathbf{C}_v)-1=0 \]

The deviatoric balance drives the viscous evolution, while the determinant constraint keeps the internal metric volume-preserving. The viscosity \(\eta\) controls the relaxation speed and hysteresis width.

5 · Consistent tangent

Implicit differentiation of the local solve

\[ \mathbf{Q}(\mathbf{F},\mathbf{h})=\mathbf{0}, \qquad \frac{d\mathbf{h}}{d\mathbf{F}} =-\mathbf{Q}_{\mathbf{h}}^{-1}\mathbf{Q}_{\mathbf{F}} \]

The local Newton problem is solved in double precision. Then the derivative of the converged history with respect to \(\mathbf{F}\) is injected into the AD variables, so the global tangent includes the hidden dependence \(\mathbf{h}(\mathbf{F})\).

6 · Weak form

Global displacement residual

\[ G_e=\int_{\Omega_e}\mathbf{P}(\mathbf{F},\mathbf{h}(\mathbf{F})): \nabla_0\delta\mathbf{u}\;dV \]

After the local update, the element contributes the same scalar weak-form potential style used by the other solid models. AD extracts the residual and the global Newton tangent from this scalar expression.

Why this belongs in the inelastic section

The response depends on the loading history and the time step. Even if the global unknown vector has only displacement DOFs, each integration point carries memory through \(\mathbf{C}_{v,n}\).

What visitors should notice

Plasticity and viscoelasticity share the same platform: declare history variables, solve a local residual, recover the consistent tangent, assemble through the generic CSR path, and plot the global response through the same postprocessing module.

1 · Global unknown

Displacement remains the only assembled field

\[ \mathbf{x}_{n+1} \equiv \mathbf{u}_{n+1} \]

The assembly asks the weak form for only the displacement field. This keeps the global CSR system identical in structure to static hyperelasticity: the additional dynamic quantities are not numbered as nodal unknowns.

2 · Local history

Velocity and acceleration live at Gauss points

\[ \mathbf{h}_{n}^{gp} = \begin{bmatrix} \mathbf{v}_{n}^{gp} \\ \mathbf{a}_{n}^{gp} \end{bmatrix} \]

In 2D, history newmark 4 means \([v_x^{gp}, v_y^{gp}, a_x^{gp}, a_y^{gp}]\) at every integration point. These values are local material-point states, not global field values.

3 · Newmark acceleration

Acceleration is reconstructed from the current displacement iterate

\[ \mathbf{a}_{n+1}^{gp} = \frac{1}{\beta \Delta t^2} \left(\mathbf{u}_{n+1}^{gp}-\mathbf{u}_{n}^{gp}-\Delta t\,\mathbf{v}_{n}^{gp}\right) - \left(\frac{1}{2\beta}-1\right)\mathbf{a}_{n}^{gp} \]

Because \(\mathbf{u}_{n+1}^{gp}\) comes from the current Newton iterate, the inertial term contributes directly to the residual and to the AD tangent.

4 · Newmark velocity

Velocity is a candidate history value

\[ \mathbf{v}_{n+1}^{gp} = \mathbf{v}_{n}^{gp} + \Delta t\left((1-\gamma)\mathbf{a}_{n}^{gp}+\gamma\mathbf{a}_{n+1}^{gp}\right) \]

The computed \(\mathbf{v}_{n+1}^{gp}\) and \(\mathbf{a}_{n+1}^{gp}\) are first written to h_trial. They are committed only when the time step is accepted.

5 · Scalar weak form

Elastic and inertial virtual work in one AD expression

\[ G(\mathbf{u}_{n+1},\delta\mathbf{u}) = \int_{\Omega_0} \mathbf{P}(\mathbf{F}(\mathbf{u}_{n+1})) : \nabla_0\delta\mathbf{u}\,dV + \int_{\Omega_0} \rho\,\mathbf{a}_{n+1}(\mathbf{u}_{n+1})\cdot\delta\mathbf{u}\,dV \]

The residual is obtained from \(\partial G/\partial\delta\mathbf{u}\), and the tangent is obtained by differentiating that residual with respect to \(\mathbf{u}_{n+1}\). Therefore the mass/inertia contribution is consistent with the Newmark discretization.

6 · Commit rule

Rejected steps must not overwrite local states

\[ \mathbf{h}_{trial}^{gp} = \begin{bmatrix} \mathbf{v}_{n+1}^{gp} \\ \mathbf{a}_{n+1}^{gp} \end{bmatrix}, \qquad \mathbf{h}_{n+1}^{gp} \leftarrow \mathbf{h}_{trial}^{gp} \quad\text{only after convergence.} \]

This is the same history-management logic used by plasticity and viscoelasticity: local trial states are allowed during Newton iterations, but the previous accepted history remains intact if adaptive stepping rejects the trial.

Theory

Continuity and momentum equations for incompressible flow.

This first fluid page stays intentionally compact. The visitor only needs to see the mixed unknowns, the incompressibility equation, and the momentum balance before reading the simulation code.

1 · Unknowns

Velocity-pressure pair

\[ \mathbf{v}(\mathbf{x},t) \in \mathbb{R}^{d}, \qquad p(\mathbf{x},t) \in \mathbb{R} \]

The input declaration uses field_key velocity pressure. Therefore the same generic element vector contains the vector velocity block and the scalar pressure block.

2 · Continuity

Incompressibility constraint

\[ \nabla \cdot \mathbf{v} = 0 \]

Pressure acts as the constraint field for this equation. In the discrete system this produces the pressure row of the mixed velocity-pressure Newton system.

3 · Momentum

Transient Navier-Stokes balance

\[ \rho \left( \frac{\partial \mathbf{v}}{\partial t} + (\mathbf{v}\cdot\nabla)\mathbf{v} \right) - \nabla\cdot\boldsymbol{\sigma} = \mathbf{0} \]

The code evaluates the time-acceleration term, the convective term, and the stress-divergence term at each Gauss point.

4 · Stress

Newtonian Cauchy stress

\[ \boldsymbol{\sigma} = -p\mathbf{I} + \mu\left(\nabla\mathbf{v}+\nabla\mathbf{v}^{T}\right) \]

The pressure part contributes -p div(delta_v), while the viscous part contributes the symmetric velocity-gradient term in the weak form.

ElementSystem compute_constitutive_element_system(
    const FieldMesh& mesh,
    int elem_id,
    const Vector& x_elem,
    const ProblemParameters& params,
    const ElementState& state
) {
    const int n = static_cast<int>(x_elem.size());

    std::vector<WeakOuterFad> x_ad(n);
    std::vector<WeakOuterFad> test_ad(n);
    for (int j = 0; j < n; ++j) {
        const WeakInnerFad x_value(x_elem[j]);
        x_ad[j] = WeakOuterFad(n, j, x_value);
    }
    for (int i = 0; i < n; ++i) {
        test_ad[i] = WeakOuterFad(WeakInnerFad(n, i, 0.0));
    }

    const WeakOuterFad G = weak_scalar_classical_fluid(mesh, elem_id, x_ad, test_ad, params, state);

    ElementSystem sys;
    sys.residual.assign(n, 0.0);
    sys.tangent.assign(n * n, 0.0);

    const WeakInnerFad G_value = G.val();
    for (int i = 0; i < n; ++i) {
        sys.residual[i] = weak_inner_dx(G_value, i);
        const int row = i * n;
        for (int j = 0; j < n; ++j) {
            const WeakInnerFad dG_dxj = (j < G.size()) ? G.dx(j) : WeakInnerFad(0.0);
            sys.tangent[row + j] = weak_inner_dx(dG_dxj, i);
        }
    }
    return sys;
}

Theory

Periodic micro fluctuation problem.

The RVE receives a macro deformation gradient and solves for a periodic fluctuation field. The averaged Piola stress is then used as the macro constitutive response.

1 · Macro input

Prescribed deformation gradient

\[\mathbf{F}_M = \begin{bmatrix} F_{11} & F_{12} \ F_{21} & F_{22} \end{bmatrix}\]

The standalone RVE run is driven by command-line or input-file values for the macro deformation gradient.

2 · Micro split

Affine part plus periodic fluctuation

\[\mathbf{u}(\mathbf{X}) = (\mathbf{F}_M-\mathbf{I})\mathbf{X} + \mathbf{w}(\mathbf{X}), \qquad \mathbf{w}^{+}=\mathbf{w}^{-}\]

Periodic boundary constraints enforce matching fluctuation values on paired RVE boundaries, while one anchor removes rigid-body motion.

3 · Homogenization

Volume-averaged Piola stress

\[\bar{\mathbf{P}} = \frac{1}{|\Omega_{RVE}|}\int_{\Omega_{RVE}} \mathbf{P}\,dV\]

The code stores both the integrated stress and the volume-normalized average, which is useful for ACEFEM-style comparisons.

4 · Sensitivity

Macro tangent data

\[\mathbb{C}_{iA aB}=\frac{\partial}{\partial F_{aB}}\int_{\Omega_{RVE}} P_{iA}\,dV\]

The sensitivity convention is integrated-Piola first; volume normalization can be applied later when needed by the macro formulation.

Theory

The macro constitutive response is computed by an RVE solve.

At each macro Gauss point, the local deformation gradient defines the micro boundary value problem. The micro response is volume-averaged and inserted into the macro weak form.

1 · Macro kinematics

Gauss-point deformation

\[\mathbf{F}_M = \mathbf{I} + \nabla_0 \mathbf{u}_M\]

The macro element computes \(\mathbf{F}_M\) exactly like a standard finite-strain solid model.

2 · RVE call

Constitutive evaluation by micro solve

\[\mathbf{F}_M \longrightarrow \text{RVE PBC solve} \longrightarrow \bar{\mathbf{P}}(\mathbf{F}_M)\]

The RVE problem replaces a closed-form stress law. This is the core macro-micro coupling point.

3 · Macro weak form

Homogenized stress enters the macro residual

\[G_M = \int_{\Omega_M} \bar{\mathbf{P}}(\mathbf{F}_M):\nabla_0\delta\mathbf{u}_M\,dV\]

The surrounding assembly, BC handling, and Newton solver remain the same as the single-scale solid path.

4 · Tangent transfer

Sensitivity supports macro Newton consistency

\[\frac{d\bar{\mathbf{P}}}{d\mathbf{F}_M}\quad \text{from RVE sensitivity}\]

The sensitivity may be obtained through the augmented RVE system and used to improve macro Newton convergence.

Theory

How the linear virtual shape function is represented without writing it explicitly.

In a polygonal VEM element, the shape functions are defined by their boundary values and polynomial consistency. The framework therefore stores vertex DOFs, reconstructs the polynomial part, and avoids requiring closed-form interior basis functions.

1 · Local space

Virtual scalar space on one polygon

\[ V_h(E)=\{v_h\in H^1(E):\; \Delta v_h=0\ \text{in}\ E,\quad v_h|_e\in\mathbb{P}_1(e)\ \forall e\subset\partial E\} \]

The boundary trace is linear on every edge, but the interior function is not written in closed form. For displacement mechanics, the vector space is simply \([V_h(E)]^2\).

2 · DOFs

Vertex values define the interpolation

\[ \mathbf{d}_E=\{u_x(\mathbf{x}_a),u_y(\mathbf{x}_a)\}_{a=1}^{n_E} \]

Linear VEM does not need midside or internal DOFs. For an \(n_E\)-sided polygon, the displacement block has \(2n_E\) local unknowns in 2D.

3 · Projected gradient

The computable part comes from a boundary integral

\[ \nabla \Pi^\nabla \mathbf{u}_h = \frac{1}{|E|}\int_{\partial E}\mathbf{u}_h\otimes\mathbf{n}\,ds \approx \frac{1}{|E|}\sum_{e=(a,b)} |e|\,\frac{\mathbf{u}_a+\mathbf{u}_b}{2}\otimes\mathbf{n}_e \]

Because the trace is linear on each edge, the edge-average value is enough. This is the main reason linear polygonal VEM can assemble a mechanical weak form without ever evaluating the unknown virtual basis function inside the polygon.

4 · Deformation

Use the projection as the polynomial strain mode

\[ \mathbf{F}_\Pi=\mathbf{I}+\nabla\Pi^\nabla\mathbf{u}_h \]

The hyperelastic stress can be evaluated from the projected deformation gradient. This gives the polynomial-consistent part of the nonlinear weak form.

5 · Fan triangles

Centroid triangulation supplies an internal integration path

\[ E=\bigcup_{e=(a,b)} T_e,\qquad T_e=\operatorname{conv}\{\mathbf{x}_c,\mathbf{x}_a,\mathbf{x}_b\} \]

The current implementation keeps a simple centroid fan. It is useful for internal quadrature, comparison against a T3-like gradient, and stabilization-style blending.

6 · Implemented scalar weak form

Projection plus triangle contribution

\[ G_E = \alpha\sum_{T\subset E}|T|\,\mathbf{P}(\mathbf{F}_T):\nabla\delta\mathbf{u}_T +(1-\alpha)|E|\,\mathbf{P}(\mathbf{F}_\Pi):\nabla\Pi^\nabla\delta\mathbf{u}_h \]

This page presents the implemented linear polygon kernel: the projected gradient carries the VEM consistency, while the fan triangles give a concrete internal contribution. The global residual and tangent are still obtained by differentiating this scalar statement with AD.

Framework concept

Local weak-form first.

The central design rule is simple: physics models are implemented at the element level, while mesh handling, global assembly, nonlinear solve, linear algebra, and output remain shared.

Local model

Element contribution

A model provides the local residual and tangent for the active fields of one element. This keeps solid, fluid, VEM, and coupled formulations isolated from the global solver.

Global reuse

Same infrastructure

After the local element system is computed, the same CSR assembly, boundary-condition handling, Newton loop, backend solver, and postprocessing path are reused.

Design benefit

Less duplicated code

Adding a model should not require rewriting how global DOFs are numbered, how matrices are assembled, or how the nonlinear solve is advanced.

Practical meaning

For a new constitutive model, the important task is to define what happens inside one element or Gauss point. The framework then lifts that local definition to a complete nonlinear finite-element problem.

Framework / Linear algebra note

Pressure Schur and PCD Approximations

This page explains the linear-algebra meaning of the pressure-block options exposed by the PETSc backend. The application pages discuss which option is appropriate for incompressible flow or FSI.

Block system

Saddle-point Newton systems

After Newton linearization and boundary-condition elimination, incompressible and coupled problems can be written in mechanical-pressure block form:

Kₘₘ Kₘₚ Kₚₘ Kₚₚ
·
Δxₘ Δp
= −
Rₘ Rₚ

Here xm collects the mechanical unknowns such as velocity, displacement, or mesh motion, while p denotes pressure. The backend does not change the weak form; it only decides how this block matrix is solved or preconditioned.

Schur complement idea

If the mechanical block is used as the primary block, the exact pressure Schur complement is

S = Kₚₚ − Kₚₘ Kₘₘ−1 Kₘₚ

Computing this exactly is expensive, so PETSc FieldSplit uses a cheaper approximation for the pressure block.

Factorization choices

  • Lower: uses a lower block-triangular Schur factorization.
  • Upper: uses an upper block-triangular Schur factorization.
  • Full: uses both block-triangular couplings for a stronger but more expensive setup.
A11 / self / full

Use existing matrix information

These options use matrix blocks already available in the Newton system. They are useful as robust baseline configurations before introducing problem-specific pressure operators.

Pressure mass

Mass-like pressure approximation

A pressure mass matrix approximates the pressure Schur block with a local, well-conditioned pressure-space operator.

Mp = ∫Ω NpT Np dΩ
Pressure Laplacian

Diffusion-like pressure approximation

A pressure Laplacian uses pressure gradients and is often useful when the Schur complement behaves like an elliptic pressure operator.

Kp = ∫Ω ∇NpT ∇Np dΩ
Mass + Laplacian

Combined pressure operator

The backend can combine mass and Laplacian terms when a single pressure operator is too weak for the current time step or Reynolds regime.

Pp = α Mp + β Kp
PCD-style option

Pressure convection-diffusion chain

The PCD-style option is a pressure-space approximation to the inverse of the pressure Schur complement S. It is not another Newton block. To keep the notation consistent with the block system above, the Newton pressure block remains Kₚₚ, while the auxiliary pressure operators are denoted by Mₚ, Lₚ, and Aₚ.

Mass Mₚ

Pressure mass operator assembled from pressure basis functions.

Mₚ = ∫Ω Nₚᵀ Nₚ dΩ
Laplacian Lₚ

Pressure Laplacian operator assembled from pressure-basis gradients.

Lₚ = ∫Ω ∇Nₚᵀ ∇Nₚ dΩ
Convection Nₚ(u)

Pressure-space convection operator evaluated with the current advective velocity.

Nₚ(u) = ∫Ω Nₚᵀ (u · ∇Nₚ) dΩ

The auxiliary pressure convection-diffusion operator is then written as

Aₚ = αₜ Mₚ + ν Lₚ + Nₚ(u)

Here αₜ represents the time/mass contribution, ν is the effective diffusion scale, and Nₚ(u) is the pressure-space convection operator. In the C++ backend, these mathematical objects are stored as Mp, Kp, and Ap inside PetscPCDData. The code name Kp corresponds to the pressure Laplacian Lₚ; it is not the Newton pressure block Kₚₚ.

Inputrₚ
Laplacian solveq = Lₚ⁻¹ rₚ
Mass actionw = Mₚ q
Transport solvezₚ = Aₚ⁻¹ w
zₚ ≈ Aₚ⁻¹ Mₚ Lₚ⁻¹ rₚ

This notation keeps the roles separate: Kₚₚ is the pressure-pressure block of the Newton matrix, S is the exact Schur complement, and Aₚ⁻¹ Mₚ Lₚ⁻¹ is the PCD-style pressure-space approximation used by the backend.

kinematic_det()

Computes the determinant of a small 2D or 3D dense matrix. In finite-strain mechanics this is the helper behind quantities such as J = det(F).

template <class Scalar>
inline Scalar kinematic_det(const std::vector<std::vector<Scalar>>& A) {
    const int n = static_cast<int>(A.size());
    if (n == 2) {
        return A[0][0] * A[1][1] - A[0][1] * A[1][0];
    }
    if (n == 3) {
        return A[0][0] * (A[1][1] * A[2][2] - A[1][2] * A[2][1])
             - A[0][1] * (A[1][0] * A[2][2] - A[1][2] * A[2][0])
             + A[0][2] * (A[1][0] * A[2][1] - A[1][1] * A[2][0]);
    }
    throw std::runtime_error("kinematic_det supports only 2D/3D matrices.");
}

kinematic_inv()

Computes the inverse of a small 2D or 3D dense matrix. This is used for deformation-gradient inverse, Piola transformations, and other tensor operations.

template <class Scalar>
inline std::vector<std::vector<Scalar>> kinematic_inv(
    const std::vector<std::vector<Scalar>>& A
) {
    std::vector<std::vector<Scalar>> B(A.size(), std::vector<Scalar>(A.size(), Scalar(0.0)));
    const Scalar detA = kinematic_det(A);

    if (A.size() == 2) {
        B[0][0] =  A[1][1] / detA;
        B[0][1] = -A[0][1] / detA;
        B[1][0] = -A[1][0] / detA;
        B[1][1] =  A[0][0] / detA;
        return B;
    }
    // 3D branch uses the analytic cofactor formula.
    return B;
}

The site shows the compact 2D branch here; the header also contains the full 3D cofactor branch.

kinematic_matmul()

Performs dense matrix multiplication for small tensors stored as row vectors. Weak forms use this for compact tensor formulas without relying on external linear algebra objects.

template <class Scalar>
inline std::vector<std::vector<Scalar>> kinematic_matmul(
    const std::vector<std::vector<Scalar>>& A,
    const std::vector<std::vector<Scalar>>& B
) {
    const int m = static_cast<int>(A.size());
    const int k = m > 0 ? static_cast<int>(A[0].size()) : 0;
    const int n = static_cast<int>(B.empty() ? 0 : B[0].size());
    std::vector<std::vector<Scalar>> C(m, std::vector<Scalar>(n, Scalar(0.0)));
    for (int i = 0; i < m; ++i)
        for (int q = 0; q < k; ++q)
            for (int j = 0; j < n; ++j) C[i][j] += A[i][q] * B[q][j];
    return C;
}

linear_projected_gradient()

Computes the constant projected gradient for a linear VEM polygon from boundary degrees of freedom. This is the key reason the method can work without explicitly constructing interior shape functions.

Projection ideagrad u ≈ (1 / area) Σ_edge u_mid ⊗ n_edge_scaled
template <class Scalar>
inline Mat2T<Scalar> linear_projected_gradient(
    const LinearElementCache& cache,
    const std::vector<Scalar>& elem
) {
    const auto& U = cache.layout.field("displacement");
    auto grad = zero_grad2<Scalar>();
    for (int ar = 0; ar < cache.n_real; ++ar) {
        const int br = (ar + 1) % cache.n_real;
        const int a_slot = cache.real_slots[ar];
        const int b_slot = cache.real_slots[br];
        for (int i = 0; i < 2; ++i) {
            const Scalar ua = elem[kinematic_local_dof_fast(U, a_slot, i)];
            const Scalar ub = elem[kinematic_local_dof_fast(U, b_slot, i)];
            const Scalar u_mid = Scalar(0.5) * (ua + ub);
            for (int A = 0; A < 2; ++A) {
                grad[i][A] += u_mid * Scalar(cache.edge_normal_scaled[ar][A] / cache.area);
            }
        }
    }
    return grad;
}

linear_triangle_gradient()

Computes a standard T3 gradient on one internal triangle of the centroid fan triangulation. This is useful for visualization, stabilization, and comparing the virtual projection with a concrete triangular submesh.

template <class Scalar>
inline Mat2T<Scalar> linear_triangle_gradient(
    const LinearElementCache& cache,
    const LinearTriangleData& tri,
    const std::vector<Scalar>& elem
) {
    const auto& U = cache.layout.field("displacement");
    auto grad = zero_grad2<Scalar>();
    for (int la = 0; la < 3; ++la) {
        const int slot = tri.slots[la];
        for (int i = 0; i < 2; ++i) {
            const Scalar ui = elem[kinematic_local_dof_fast(U, slot, i)];
            for (int A = 0; A < 2; ++A) {
                grad[i][A] += ui * Scalar(tri.dN[la][A]);
            }
        }
    }
    return grad;
}
